# 07. 營運計劃

## 7.1 組織架構

### Phase 1（Month 1~6）：精兵期
```
創辦人（CEO + 產品）
    ├── 工程師（兼職/外包）
    └── 行銷（兼職）

創辦人（CEO）

```

## 7.2 導遊審核 SOP

### 申請流程
1. 導遊填寫申請表（/guide/apply）
2. 上傳文件：
   - 身分證（正反面）
   - 導遊執照（如有）
   - 個人照片
   - 自我介紹影片（30 秒，可選）
3. 系統通知管理員審核
4. 審核時間：2~3 個工作天
5. 通過 → 寄發歡迎信 + 引導上架第一個活動
6. 退件 → 說明原因 + 補件指引

### 審核標準
| 項目 | 必填 | 說明 |
|------|------|------|
| 身分證 | ✅ | 本名驗證 |
| 銀行帳號 | ✅ | 結算用 |
| 個人照片 | ✅ | 消費者信任感 |
| 導遊執照 | ✅ 優先 | 無證者需更詳細審核 |
| 個人簡介 | ✅ | 至少 100 字 |

### 拒絕條件
- 身分驗證失敗
- 有違法或詐騙紀錄
- 無法提供真實聯繫方式

## 7.3 客服 SOP

### 客服管道
- 主要：官網 Chat（Crisp 或 Intercom）
- 輔助：Email（support@[domain].com）
- 緊急：LINE Official Account

### 回應時間標準
| 管道 | 目標回應時間 |
|------|------------|
| Chat（上班時間） | < 5 分鐘 |
| Email | < 24 小時 |
| 緊急（活動當天） | < 30 分鐘 |

### 常見問題處理

**Q：我想取消訂單**
→ 確認訂單日期，按退款政策執行，若符合條件自動退款

**Q：導遊沒出現 / 行程取消**
→ 緊急處理，全額退款 + 補貼 NT$100 + 道歉信

**Q：導遊的服務不符預期**
→ 記錄投訴，調查後若屬實扣導遊信用分，嚴重者停權

**Q：我想成為導遊**
→ 引導填寫申請表，提供申請說明文件

## 7.4 活動品質控制

### 活動上架審核
- 標題：不得含誤導性詞彙
- 照片：至少 3 張真實照片（不得使用 Stock 圖）
- 描述：至少 200 字，清楚說明行程內容
- 定價：不得低於 NT$300 / 人

### 定期品質檢查（每月）
- 評分 < 3.5 的導遊 → 發送改善通知
- 連續 2 個月 < 3.5 → 暫停上架，要求面談
- 連續 3 個月 < 3.5 → 停權

### 黑名單機制
- 遊客：多次惡意退款者、評價騷擾者
- 導遊：詐欺、不實描述、無故爽約

## 7.5 緊急應變計劃

### 技術事故（服務中斷）
1. Vercel / Supabase 狀態頁監控
2. 事故發生 → 10 分鐘內發布狀態更新
3. 若影響訂單 → 主動聯繫受影響用戶
4. 事故報告（Post-mortem）在 48 小時內完成

### 金流事故
1. ECPay 回報異常 → 立即暫停接單
2. 聯繫 ECPay 商家支援
3. 告知等待中的消費者延遲說明
4. 備援：藍新金流切換（需提前申請）

### 安全事故（資料洩漏）
1. 立即下線受影響服務
2. 24 小時內通報（個資法規定）
3. 通知受影響用戶
4. 法律顧問介入

## 7.6 KPI 監控儀表板

- GMV
- 新導遊申請數
- 評價平均分數
- 活躍活動數

### 每月追蹤
- MRR（月重複收入）
- CAC（獲客成本）
- Churn Rate（流失率）
- NPS（淨推薦值）
