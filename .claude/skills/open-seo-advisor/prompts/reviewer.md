# Reviewer Prompt（審核把關）

在任何模式產出最終報告、Patch Plan 或內容草稿交給使用者之前，執行以下
自我審核（對應整體工作流程中的「審核端」把關角色）：

## 通用審核項目

1. **證據充分性**：每個 Finding / 修改建議 / 內容主張，是否都有具體
   證據支撐（HTTP 回應、HTML 內容、檔案內容），而非憑空推論？
2. **過度推論檢查**：是否有把「可能」講成「一定」的地方？
   信心不足的判斷是否已標示 `confidence` 較低？
3. **授權邊界檢查**：所有操作是否都在使用者明確授權的範圍內？
   是否有任何步驟嘗試存取未授權的系統或資料？
4. **安全邊界檢查**：是否有任何寫入/部署動作繞過了 dry-run 或人工確認？
   是否有憑證或敏感資訊意外出現在輸出內容中？
5. **可執行性檢查**：修復建議是否具體到工程師可以直接動手做，而不是
   空泛的「建議優化 XXX」？
6. **一致性檢查**：報告中的優先順序、嚴重度是否與 `config/scoring.yaml`
   的定義一致？

若任何一項未通過，應在最終輸出前修正，或明確向使用者說明限制與風險，
而非隱瞞不確定性。
