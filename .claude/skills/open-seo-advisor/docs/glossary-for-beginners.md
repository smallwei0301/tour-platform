# SEO 術語白話對照表（給完全不懂技術的人看）

看不懂技術報告（`report.md`）裡的術語嗎？這份對照表用生活化比喻解釋每個常見詞彙。

---

## Sitemap（網站地圖）

**白話**：就像百貨公司的樓層導覽圖，把「這個網站有哪些頁面」列給搜尋引擎看。

**為什麼重要**：如果地圖漏掉重要樓層（頁面），逛街的人（搜尋引擎）可能要花很久才找得到，甚至永遠找不到。

## Robots.txt

**白話**：貼在店門口的「哪些區域歡迎參觀、哪些是員工專區」告示牌。

**為什麼重要**：設定錯誤可能會不小心把「歡迎參觀」的重要頁面標成「員工專區」，導致搜尋引擎不敢進去看。

## Canonical（正規網址）

**白話**：告訴搜尋引擎「這個商品雖然在很多樓層都看得到，但這一份型錄才是正式版本」。

**為什麼重要**：如果同一件商品有好幾份型錄卻沒說明哪份是正式的，搜尋引擎可能不知道該推薦哪一個，甚至兩份都不推薦。

## Title（標題標籤）

**白話**：頁面的「店名招牌」，會直接顯示在 Google 搜尋結果的藍色標題上。

**為什麼重要**：招牌寫得不清楚或每間店都掛一樣的招牌，客人（使用者）就很難分辨要走進哪一間。

## Meta Description（描述標籤）

**白話**：招牌下面那行「本店專賣...」的介紹文字，會出現在搜尋結果的標題下方。

**為什麼重要**：雖然不會直接影響排名，但寫得吸引人，路過的人比較願意點進來看。

## H1（主標題）

**白話**：走進店裡後看到的「主要招牌」，告訴客人這一區在賣什麼。

**為什麼重要**：如果沒有主招牌，或掛了好幾個互相矛盾的主招牌，客人會搞不清楚重點。

## 內部連結（Internal Link）

**白話**：店內的指標牌，引導客人從一區走到另一區。

**為什麼重要**：如果某個區域完全沒有指標牌指過去（孤兒頁），客人和搜尋引擎都可能永遠找不到那裡。

## 孤兒頁（Orphan Page）

**白話**：店裡有一區沒有任何指標牌指向它，只能靠客人自己亂逛才會發現。

**為什麼重要**：搜尋引擎主要靠連結探索頁面，沒有連結指向的頁面很難被發現與收錄。

## 結構化資料（Structured Data / Schema）

**白話**：附在商品旁邊、給機器人看得懂的規格表（品名、價格、評分），讓搜尋引擎能直接在搜尋結果秀出星等、價格等資訊。

**為什麼重要**：規格表寫得完整，搜尋結果可能會有更豐富的呈現方式（例如星星評分），提高吸引力。

## Core Web Vitals（核心網頁指標）

**白話**：客人從走進店門到真正能開始逛、能順暢移動、畫面不會一直跳動的「體驗流暢度」評分。

**為什麼重要**：載入太慢、畫面一直跳動，客人容易不耐煩離開，Google 也會把這件事納入排名考量。

## hreflang（多語言標記）

**白話**：多語言分店之間互相標註「這是中文版、這是英文版」的告示牌。

**為什麼重要**：沒標清楚的話，搜尋引擎可能把英文版推薦給只看得懂中文的客人。

## E-E-A-T（經驗、專業、權威、可信）

**白話**：判斷「這間店的老闆看起來專業嗎、講的話可信嗎、有沒有實際經驗」的綜合印象分。

**為什麼重要**：內容如果缺乏真實經驗、專業背景或可信來源，搜尋引擎與使用者都比較不容易信任。

## 優先順序（P0 / P1 / P2 / P3）是什麼？

我們用「房屋健檢」的比喻來分級：

| 等級 | 白話說明 | 比喻 |
|---|---|---|
| P0 | 今天就要處理 | 家裡漏水或跳電，已經影響基本功能 |
| P1 | 這週要處理 | 大門門鎖故障或電線老舊，還能用但風險高 |
| P2 | 排進近期改善 | 屋內動線或收納不順，體驗會被拖累 |
| P3 | 有空再優化 | 牆面掉漆、擺設可以更好看，不影響基本運作 |

還有看不懂的詞嗎？歡迎到專案的 GitHub Issues 提出，我們會持續擴充這份對照表。
