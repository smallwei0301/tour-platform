## 📋 核心法規框架

本平台（導遊媒合 B2C 電商）涉及三大法規體系：

| 法規 | 主管機關 | 關鍵影響 | 風險等級 |
|------|---------|---------|---------|
| 旅行業管理規則 + 發展觀光條例 | 交通部觀光署 | 是否需旅行業執照 | 🔴 高 |
| 消費者保護法（CPA） | 行政院消保會 | 七日鑑賞期、退款義務 | 🟡 中 |
| 個人資料保護法（PDPA 2025修正版） | 個資保護委員會（PDPC） | 個資蒐集、資安、違規通報 | 🟡 中 |
| 電子商務消費者保護綱領 | 行政院 | B2C 電商強制規範 | 🟡 中 |
| 稅務（加值稅/所得稅） | 財政部 | 電子發票、預扣稅 | 🟢 低 |

## 一、旅行業執照合規

### 1.1 平台定性分析

**問題核心：** 導遊媒合平台是否構成「旅行業」？

**台灣旅行業定義**（發展觀光條例第2條）：
> 旅行業 = 「以營利為目的，提供旅遊行程、住宿、導遊，或代購票券、辦理旅遊文件之事業」

**平台模式分析：**

| 業務項目 | 我們做 | 旅行業範疇 |
|---------|------|----------|
| 安排行程（套裝） | ❌ 否 | ✅ 是旅行業 |
| 媒合導遊 + 消費者 | ✅ 是 | ⚠️ 灰色地帶 |
| 代收款項（代金流） | ✅ 是 | ⚠️ 爭議點 |
| 提供導遊資料查詢 | ✅ 是 | ❌ 非旅行業 |
| 保險安排（旅遊險） | ❌ 否（自選） | ✅ 是旅行業 |

**初步結論：** 純媒合 + 資訊揭露模式（不包裝行程）屬**灰色地帶**；若平台收取服務費並作為金流中介，主管機關（觀光署）可能認定須申請旅行業執照。

### 1.2 乙種旅行業執照路線（建議）

若決定申請，建議走**乙種旅行業（Class-B）**：

**條件：**
- 最低資本額：NTD 600,000
- 保證金：NTD 1,500,000（存入觀光署指定銀行）
- 公司名稱需含「旅行社」字樣
- 需聘有執照之旅行業經理人

**業務範圍（乙種）：**
- ✅ 國內旅遊安排
- ✅ 媒合導遊（本國人）
- ❌ 不可接待國際觀光客

> 若要接待外國旅客，需升級為**甲種或綜合旅行業**（資本額 NTD 3,000,000 / NTD 6,000,000）

### 1.3 短期規避策略（MVP 期間）

在取得執照前，可採用以下**合規降風險**方式：

1. **信息媒合模式**：平台只提供導遊資訊，不介入金流（消費者自行付款給導遊）
2. **委任關係聲明**：明確聲明平台為「科技媒合工具」而非旅行業者
3. **諮詢主管機關**：向觀光署申請預先諮詢（免費），確認平台定性
4. **法律顧問意見書**：建議取得一份台灣旅遊法專業律師意見書（費用約 NTD 30,000-60,000）

## 二、消費者保護法合規

### 2.1 七日鑑賞期（猶豫期）

**CPA 第19條核心規定：**
- B2C 網路交易：消費者有 **7 天猶豫期**，無需理由，可免費退訂
- 退款時限：賣方需在 **收到通知後 15 天內** 完成退款
- 消費者無需支付任何費用

**旅遊服務例外（重要！）：**

| 情況 | 是否適用7日猶豫 |
|------|--------------|
| 一般商品購買 | ✅ 完全適用 |
| 旅遊服務（有明確日期） | ⚠️ 縮短至「服務前 N 天」 |
| 已提供之服務 | ❌ 不適用 |
| 導遊行程預訂（未開始） | ✅ 適用，但有標準契約限制 |

### 2.2 旅遊定型化契約（強制）

交通部觀光署已公告**旅遊定型化契約應記載及不得記載事項**：

**必須記載：**
- 行程內容、費用明細
- 退費計算方式（依距出發日天數遞增違約金比例）
- 緊急聯絡方式
- 旅遊糾紛申訴管道

**退費標準（強制比例）：**

| 取消距出發日 | 消費者可退金額 |
|------------|-------------|
| 31 天以上 | 退 100%（或扣行政費 5%） |
| 21-30 天 | 退 80% |
| 11-20 天 | 退 70% |
| 5-10 天 | 退 60% |
| 4 天以下 | 退 30-50% |
| 當日/不來 | 退 0-30% |

> ⚠️ **注意**：我們在 07-operations-plan 承諾「3天退款」是指**退款處理速度**，不是退費比例 — 退費比例仍需依上表，不可比消保法規定對消費者更不利。

### 2.3 電子商務強制揭露事項

平台網站必須清楚標示：

- [ ] 公司名稱、負責人、地址
- [ ] 統一編號（公司登記號碼）
- [ ] 客服聯絡方式（電話/Email）
- [ ] 隱私政策連結
- [ ] 服務條款連結
- [ ] 退款政策（CPA 合規版）
- [ ] 付款方式說明
- [ ] 爭議解決管道

## 三、個人資料保護法（PDPA）合規

### 3.1 2025 修正版重點（2025-11-11 公告）

**新增義務（對平台影響）：**

| 新規定 | 影響 | 因應措施 |
|--------|------|---------|
| 設立個資保護委員會（PDPC）| 新主管機關 | 關注 PDPC 正式運作公告 |
| 資料外洩強制通報（PDPC + 當事人） | 需建立事件應變流程 | 建立 incident response SOP |
| 強化安全措施義務（§20-1）| 需建立個資檔案安全計畫 | 文件化資安政策 |
| 違規罰鍰 | NTD 20,000–200,000 + 每次拒絕改正再罰 | 確保通報機制 |

### 3.2 平台個資蒐集清單

| 資料類型 | 蒐集對象 | 法律依據 | 保存期限 |
|---------|---------|---------|---------|
| 姓名、Email、電話 | 旅客、導遊 | 契約執行 | 合約存續 + 2年 |
| 支付資訊（信用卡後四碼） | 旅客 | 契約執行 | 依金流服務商規定 |
| 身分驗證文件（導遊執照） | 導遊 | 法令義務（KYC） | 合約期間 |
| 行程評論、評分 | 旅客 | 正當利益 | 無限期（公開） |
| IP 位址、裝置資訊 | 全用戶 | 正當利益（資安）| 1 年 |
| GPS/位置資訊 | 旅客（選擇性） | 明確同意 | 使用後即刪 |

### 3.3 隱私政策必要事項（§8 告知義務）

用戶同意蒐集個資前，必須告知：
1. 蒐集機關名稱
2. 蒐集目的
3. 個資類別
4. 利用期間、地區、對象及方式
5. 當事人權利（查閱、更正、刪除、停止）
6. 不提供資料之影響

### 3.4 跨境資料傳輸

若使用 Supabase（美國伺服器）或 Vercel：
- 需在隱私政策中告知「資料可能傳輸至台灣以外地區」
- 確保資料處理服務商有充分個資保護（DPA 條款）

## 四、KYC / 導遊身份驗證合規

### 4.1 強制驗證義務

依旅行業管理規則，執業導遊需具備：
- 交通部觀光署核發的**導遊執照**（定期更新）
- 或**領隊執照**（帶出境旅遊）

**平台責任：** 在媒合前需驗證導遊資格有效性

**驗證 SOP：**
1. 導遊上傳執照影本
2. 系統記錄執照號碼 + 到期日
3. 定期提醒導遊更新（到期前 30 天）
4. 暫停已過期導遊的接單資格

### 4.2 實名制要求

- 導遊：需提供身分證 + 導遊執照（強制）
- 旅客：依風險等級 — 低風險（Email + 電話即可），高金額交易建議加強驗證

## 五、稅務合規

### 5.1 電子發票義務

- 銷售額達門檻需開立**統一發票**（電子發票）
- 平台服務費（抽成）需開發票給導遊（B2B）
- 旅客支付的行程費用：若平台為代收方，視為代理，稅務處理需明確

### 5.2 預扣稅（Withholding Tax）

- 支付給**外籍導遊**：需預扣 20% 所得稅
- 支付給**本國導遊**：超過 NTD 20,000/月需申報扣繳

### 5.3 加值稅（VAT / 5%）

- 導遊服務費：需含 5% 加值稅
- 平台服務費：需含 5% 加值稅
- 跨境服務（外國旅客買台灣行程）：可能適用零稅率，建議諮詢稅務顧問

## 六、合規行動清單（Compliance Checklist）

### Phase 1：MVP 上線前（必做）

- [ ] 完成公司設立（含旅行業或科技媒合業定性諮詢）
- [ ] 起草隱私政策（PDPA 合規版）
- [ ] 起草服務條款（消保法合規 + 退費條款）
- [ ] 起草旅遊定型化契約（依交通部公告版本）
- [ ] 導遊入駐 KYC 流程設計（執照驗證）
- [ ] 電子發票系統串接（或委外代開）
- [ ] 建立個資外洩事件應變 SOP（PDPA §12 要求）

### Phase 2：正式營運後 3 個月內

- [ ] 觀光署預先諮詢（確認平台定性）
- [ ] 取得法律顧問意見書（旅遊媒合平台合規）
- [ ] 申請乙種旅行業執照（如主管機關要求）
- [ ] 建立消費者客訴處理機制（30分鐘回應 SLA）
- [ ] 稅務顧問確認服務費代收模式

### Phase 3：成長期

- [ ] ISO 27001 資安認證（個資保護加分）
- [ ] 旅遊保險合作夥伴引入（降低法律風險）
- [ ] 定期 PDPA 合規審計（每年一次）

## 七、主要風險與緩解策略

| 風險 | 機率 | 影響 | 緩解策略 |
|------|------|------|---------|
| 觀光署認定需旅行業執照 | 中 | 高 | 預先諮詢；初期走資訊媒合模式 |
| 消費者投訴退款糾紛 | 高 | 中 | 清楚揭露退費政策；快速處理 |
| 個資外洩被裁罰 | 低 | 高 | 建立 PDPA 合規計畫；ISO 27001 |
| 導遊執照驗證失敗（假執照）| 低 | 高 | API 對接觀光署查詢系統 |
| 稅務申報錯誤 | 中 | 中 | 委外記帳士 + 稅務顧問 |
