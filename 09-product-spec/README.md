## 一、User Stories（完整版）

### 消費者（遊客）

```
[信任與驗證]
AS A 遊客
I WANT TO 看到導遊的真實身份驗證標章
SO THAT 我知道這個人是真實存在且可信任的

[退款透明]
AS A 遊客
I WANT TO 在訂購前就清楚看到退款政策
SO THAT 我不會因為找不到細則而被坑

[退款追蹤]
AS A 已申請退款的遊客
I WANT TO 追蹤退款進度
SO THAT 我不需要一直發信問「退款到了嗎」

[緊急聯繫]
AS A 活動當天遇到問題的遊客
I WANT TO 一鍵找到緊急聯繫管道
SO THAT 我不會在現場不知所措

[替代選項]
AS A 被告知訂單取消的遊客
I WANT TO 立刻看到 3 個同類替代活動
SO THAT 我的旅遊計劃不會完全泡湯

[評價]
AS A 完成活動的遊客
I WANT TO 留下真實評價
SO THAT 我可以幫助其他遊客選擇

[收藏]
AS A 瀏覽中的遊客
I WANT TO 收藏喜歡的活動
SO THAT 之後可以回來查看
```

### 導遊

```
[接案管理]
AS A 導遊
I WANT TO 在後台集中管理所有訂單
SO THAT 我不用靠 LINE 群管理客人

[排班控制]
AS A 導遊
I WANT TO 設定哪些日期可預約、哪些已滿
SO THAT 我不會被雙重訂單困住

[收益透明]
AS A 導遊
I WANT TO 即時看到我的收益明細和預計入帳時間
SO THAT 我知道錢在哪、什麼時候到

[提款]
AS A 導遊
I WANT TO 申請提款並選擇銀行帳號
SO THAT 我可以把平台收益轉到自己帳戶

[行前準備]
AS A 導遊
I WANT TO 在活動前一天發送提醒給報名遊客
SO THAT 遊客知道集合地點和注意事項

[評價回應]
AS A 導遊
I WANT TO 回覆遊客的評價
SO THAT 我可以說明特殊情況或感謝好評
```

## 二、核心功能規格（v2）

### F00 — 信任系統（新增）

**導遊驗證標章顯示規則：**
- 未驗證：顯示「未驗證」灰色標籤
- 完成身分證 KYC：顯示「✅ 已實名認證」
- 具備導遊執照：額外顯示「📋 持照導遊」
- 精選導遊：顯示「⭐ 平台精選」

**評價真實性標示：**
- 每則評價顯示「已驗證訂單」標章
- 評價日期 + 行程名稱公開顯示

---

### F01 — 首頁

- 三條信任承諾橫幅（退款 3 天 / 緊急熱線 / 實名驗證）
- 搜尋列：輸入地區或活動類型即可篩選
- 精選導遊輪播（⭐ 精選導遊）
- 熱門行程分類卡片（台北 / 台中 / 台南 / 花東 / 其他）
- 最新評價牆（5 則真實評價）

---

### F02 — 活動列表頁（升級）

篩選條件（新增）：
- ✅ 僅顯示「已驗證導遊」
- ✅ 僅顯示「持照導遊」
- 語言：中文 / 英文 / 日文 / 韓文導遊
- 人數：1人 / 2~4人 / 5人以上
- 活動長度：半天 / 一天 / 多天

---

### F03 — 活動詳情頁（升級）

退款政策區塊：
- **顯著位置**（不藏在細則）
- 用顏色表格顯示各時段退款比例
- 「我了解退款政策」勾選框（付款前必勾）

導遊資訊區塊（擴充）：
- 自我介紹影片（30 秒）
- 驗證標章群組
- 評價分布圖（1~5 星各有幾則）
- 最近評價 5 則（可展開全部）
- 「問導遊」按鈕（訂購前詢問）

---

### F04 — 訂單管理與退款追蹤（新增）

消費者訂單頁新增：
- 退款狀態進度條：`申請中 → 審核中 → 處理中 → 已入帳`
- 預計入帳日期顯示
- 每個狀態變更時發送 Email + LINE 通知
- 緊急聯繫按鈕（活動前 24h 至活動結束後 2h 才顯示）

---

### F05 — 緊急求助系統（新增）

觸發條件：訂單活動日期當天

功能：
- 一鍵撥打客服緊急電話
- 一鍵傳送 LINE 給客服（預填訂單資訊）
- 顯示集合地點 Google Maps（方便確認位置）
- 導遊手機號碼（訂單確認後才顯示）

---

### F06 — 導遊後台（升級）

Dashboard 新增：
- 本月收益圖表（預估 vs 已確認）
- 即將到來行程清單（7 天）
- 需要確認的訂單提醒（紅色標示）

訂單管理：
- 接受 / 婉拒 待確認訂單（婉拒需填原因）
- 婉拒時系統自動通知遊客並推薦替代活動

行前提醒工具：
- 選擇訂單 → 編輯提醒內容 → 一鍵傳送給所有報名者
- 預設模板（集合地點、建議穿著、注意事項）

## 三、頁面地圖（v2）

```
/ (首頁)
├── /activities (活動列表)
│   └── /activities/[id] (活動詳情)
├── /guides (導遊列表)
│   └── /guides/[id] (導遊個人頁)
├── /booking/[activityId] (預約流程)
├── /orders (我的訂單)
│   └── /orders/[id] (訂單詳情 + 退款追蹤)
├── /emergency (緊急求助 — 活動當天顯示)
├── /auth (登入/註冊)
├── /guide
│   ├── /guide/apply (申請)
│   └── /guide/dashboard (後台)
│       ├── activities (活動管理)
│       ├── orders (訂單管理)
│       ├── earnings (收益 + 提款)
│       └── tools (行前提醒工具)
└── /admin (平台後台)
    ├── guides (導遊審核)
    ├── orders (爭議訂單)
    ├── refunds (退款管理)
    └── reports (數據報表)
```

## 四、技術實作優先序（MVP vs Later）

| 功能 | 優先序 | 說明 |
|------|--------|------|
| 信任標章系統 | ⭐ MVP | 核心差異化，Day 1 就要有 |
| 退款政策顯示（明顯位置） | ⭐ MVP | 解決 KKday 最大痛點 |
| 退款進度追蹤 | ⭐ MVP | 解決第二大痛點 |
| 評價（驗證訂單才能寫）| ⭐ MVP | 信任系統基礎 |
| 緊急聯繫頁面 | ⭐ MVP | 活動當天必要功能 |
| 導遊自介影片 | ⭐ MVP | 差異化最強的功能 |
| 替代活動推薦（取消時）| Phase 2 | 需要算法支援 |
| 行前提醒工具 | Phase 2 | 導遊體驗提升 |
| 多語言介面 | Phase 2 | 外籍遊客市場 |
| 企業客戶功能 | Phase 3 | B2B 功能 |
