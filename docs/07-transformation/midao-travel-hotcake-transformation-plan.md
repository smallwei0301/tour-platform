# 祕島轉型計畫：旅遊業 Hotcake 免費導遊管理工具 + 前台曝光商業模式

> 版本代碼：`MIDAO-HOTCAKE-PLAN-2026-06-16-V1`
> 建立日期：2026-06-16
> 文件目的：把「祕島」從旅遊媒合平台，規劃成旅遊業者與導遊可免費使用的預約管理 SaaS，並在供給與流量成長後，加入前台曝光、推薦排序與主題專區等廣告收入。

---

## 1. 核心結論

祕島最值得採用的 10/10 轉型路線不是一開始就做旅遊媒合平台，也不是一開始就賣首頁廣告，而是先做：

> 給導遊、在地體驗主理人與小型旅遊業者使用的免費預約管理工具。

再逐步變成：

> 旅遊業 Hotcake：幫導遊管理行程、時段、訂單、旅客名單、通知、行銷、報表與平台曝光。

此策略的關鍵是：

1. 祕島目前沒有既有瀏覽量，因此「首頁曝光」初期不能作為主要收入。
2. 導遊真正會立即需要的是免費預約頁、行程管理、訂單管理、客戶名單與行事曆。
3. 導遊如果願意把自己的 IG、LINE、熟客導入祕島預約頁，祕島就能用供給方反向帶動流量。
4. 等平台累積導遊數、行程數、SEO 頁面與旅客瀏覽量後，再販售前台曝光會更合理。
5. 收入順序應該是「工具付費功能」先於「廣告曝光」。

簡單比喻：

> 第一階段不是賣百貨公司櫥窗，而是先免費送導遊一台好用的預約本與收銀機。等很多導遊與客人都進來之後，祕島的櫥窗才有價格。

## 2. 產品定位

### 2.1 對外一句話定位

> 祕島是給導遊與在地體驗業者使用的免費預約管理系統，幫你建立專業行程頁、管理預約、整理客戶，並有機會被更多旅客看見。

### 2.2 不建議的定位

不建議初期定位為：

- 台灣最大旅遊平台
- 導遊版 Klook
- 旅遊廣告曝光平台
- 付費首頁推薦平台

原因是祕島目前沒有足夠流量，若直接賣曝光，導遊會要求證明曝光能帶來訂單。

### 2.3 建議定位

建議初期定位為：

- 導遊免費預約系統
- 在地體驗業者免費接單工具
- 私人導遊與秘境行程的專業預約頁
- 旅遊業者的客戶管理與行事曆工具

## 3. 商業模式順序

### 3.1 第一階段：免費工具取得導遊供給

提供免費功能：

- 導遊公開頁
- 最多 3 個行程
- 基礎預約頁
- 基礎行事曆
- 基礎訂單管理
- 客戶名單 80 位
- 每月 30 筆預約
- 基礎 Email 通知
- 基礎報表

此階段目標不是賺錢，而是讓導遊願意把自己的旅客帶進祕島。

### 3.2 第二階段：工具進階功能收費

優先收費功能：

- LINE 自動提醒
- Google Calendar 同步
- 客戶資料匯出
- 優惠碼
- 訂金 / 尾款提醒
- 進階報表
- 自訂品牌頁
- 多導遊帳號
- 多語系頁面
- 包團詢價表

這些功能與平台流量無關，即使祕島還沒有大流量，也有機會向導遊收費。

### 3.3 第三階段：前台曝光與廣告收入

等平台有可證明流量後，再推出：

- 首頁精選曝光
- 分類頁置頂
- 搜尋排序加權
- 主題專區推薦
- 新行程推薦
- 季節活動檔期曝光

曝光產品必須搭配數據報表：

- 曝光數
- 點擊數
- 收藏數
- 詢問數
- 預約數
- 成交金額

否則導遊無法判斷廣告是否值得續費。

## 4. 免費版、付費版與曝光方案設計

| 方案 | 目標對象 | 建議價格 | 核心功能 |
|---|---|---:|---|
| 免費版 | 個人導遊、新手體驗業者 | NT$0 / 月 | 3 個行程、30 筆預約/月、80 位客戶、基礎行事曆、基礎訂單 |
| 入門版 | 穩定接案導遊 | NT$399～699 / 月 | 10 個行程、200 筆預約/月、500 位客戶、LINE 提醒、優惠碼、客戶匯出 |
| 專業版 | 導遊團隊、小型工作室 | NT$990～1,990 / 月 | 多導遊、多語系、包團詢價、訂金尾款、進階報表、客戶分眾 |
| 曝光加值包 | 需要平台導流的導遊 | 依流量與版位定價 | 首頁精選、分類頁置頂、搜尋加權、主題專區 |

### 4.1 免費版設計原則

免費版要足夠好用，讓導遊願意真的使用；但要限制會提高成本或高商業價值的功能。

免費版可以給：

- 基礎資料儲存
- 基礎公開頁
- 少量行程
- 少量預約
- 少量客戶名單

免費版應限制：

- LINE 通知
- 簡訊
- 大量圖片
- 客戶匯出
- 進階報表
- 自訂網址
- 多帳號權限
- 曝光版位

## 5. 導遊後台對標 Hotcake 的功能藍圖

祕島若要成為旅遊業 Hotcake，導遊後台不能只是上架行程，而是要成為導遊每天工作的核心工具。

- 今日預約
- 本週出團

- 待回覆詢問
- 本月營收
- 本月預約數

- 曝光成效

- 登山嚮導
- 包車導覽

核心欄位：

- 行程名稱

- 難度

## 6. 前台曝光產品設計

前台曝光不能只賣「首頁位置」，應該拆成不同旅客意圖。

### 6.1 首頁精選

適合：

- 品牌導遊
- 高品質行程
- 季節主打活動

風險：

- 初期無流量時價值低
- 若廣告過多會降低旅客信任

### 6.2 分類頁置頂

適合：

- 地區型行程，例如花蓮、台東、澎湖
- 類型行程，例如溯溪、SUP、城市散步

這通常比首頁曝光更精準，建議優先發展。

### 6.3 搜尋排序加權

排序不應完全由付費決定，建議公式：

```txt
推薦分數 = 品質分數 + 評價分數 + 回覆速度 + 可預約程度 + 付費曝光加權
```

### 6.4 主題專區

適合 SEO 與檔期行銷：

- 夏天玩水專區
- 外國人友善導覽
- 親子秘境行程
- 雨天也能玩的體驗
- 企業 Team Building
- 台灣離島深度旅行

## 7. 資料庫設計方向

祕島建議採用多租戶 SaaS 思維。白話說，就是所有導遊共用一套資料庫，但每筆資料都要標記屬於哪一位導遊或哪一個團隊。

### 7.1 核心資料概念

建議核心資料模型：

```txt
guide_profiles / provider_profiles
  ├─ guide_team_members
  ├─ activities
  │    └─ activity_plans
  ├─ availability_rules
  ├─ blackout_dates
  ├─ bookings
  │    ├─ booking_participants
  │    └─ booking_status_events
  ├─ orders
  │    ├─ payments
  │    └─ refunds
  ├─ traveler_contacts
  │    ├─ traveler_tags
  │    └─ traveler_notes
  ├─ marketing_campaigns
  ├─ exposure_campaigns
  │    └─ exposure_events
  └─ subscriptions
```

### 7.2 權限隔離原則

所有導遊後台資料都應能用以下概念隔離：

- `guide_id`
- `provider_id`
- `team_id`
- `owner_user_id`

避免 A 導遊看到 B 導遊的客戶資料。

### 7.3 成本控制資料設計

免費版限制應該從資料層與應用層同時控制：

- 每位導遊最多 3 個行程
- 每月最多 30 筆預約
- 最多 80 位旅客 CRM 聯絡人
- 圖片張數與容量限制
- 報表查詢時間範圍限制
- 通知發送量限制

## 8. 10/10 執行計畫

### Phase 0：定位驗證，1～2 週

目標：確認導遊真的願意使用免費工具。

工作項目：

1. 訪談 15～30 位導遊與體驗業者。
2. 確認他們目前如何接單：LINE、IG、Google 表單、Klook、KKday、旅行社。
3. 確認最痛的問題：漏單、改期、收訂金、旅客資料分散、行程頁不專業。
4. 做一頁式 Landing Page，收集導遊申請名單。
5. 手動幫 5～10 位導遊建立示範行程頁。

成功指標：

- 至少 10 位導遊願意提供行程資料。
- 至少 5 位導遊願意把祕島連結放到 IG / LINE。
- 至少 3 位導遊願意用祕島接真實預約。

### Phase 1：免費導遊工具 MVP，3～5 週

目標：讓導遊可以真的用祕島管理預約。

必做功能：

1. 導遊公開頁。
2. 行程建立與編輯。
3. 預約頁。
4. 訂單列表。
5. 訂單詳情。
6. 基礎行事曆。
7. 旅客名單。
8. Email 通知。
9. 免費方案限制。

成功指標：

- 30 位導遊註冊。
- 100 個行程上架。
- 50 筆真實預約或詢問。
- 20% 導遊一週內回訪後台。

### Phase 2：工具付費功能，4～6 週

目標：開始驗證導遊願不願意為效率工具付費。

必做功能：

1. LINE 預約提醒。
2. Google Calendar 同步。
3. 優惠碼。
4. 客戶資料匯出。
5. 訂金 / 尾款提醒。
6. 進階品牌頁。
7. 基礎訂閱方案。

成功指標：

- 5～10 位付費導遊。
- 免費轉付費率達 5% 以上。
- 付費導遊月留存 70% 以上。

### Phase 3：SEO 與前台內容成長，8～12 週

目標：讓祕島開始有自然流量。

工作項目：

1. 建立地區頁：花蓮、台東、澎湖、台北、台南、金門、馬祖。
2. 建立主題頁：溯溪、SUP、登山、城市散步、親子、外國人導覽。
3. 每個行程頁加入結構化資料。
4. 建立導遊故事與在地玩法文章。
5. 建立內部連結：地區頁、主題頁、導遊頁、行程頁互相串接。

成功指標：

- 每月自然搜尋流量穩定成長。
- 地區 / 主題頁開始帶來詢問。
- 每個月新增 50～100 個可索引行程頁。

### Phase 4：曝光產品 Beta，4～6 週

目標：開始測試前台曝光是否能帶來可衡量成效。

必做功能：

1. 曝光活動管理。
2. 曝光事件追蹤。
3. 點擊事件追蹤。
4. 詢問 / 預約轉換追蹤。
5. 導遊曝光成效報表。
6. 分類頁置頂 Beta。
7. 主題專區 Beta。

成功指標：

- 曝光頁點擊率可被追蹤。
- 至少 3 位導遊願意購買或續用曝光。
- 曝光導流能產生詢問或預約。

### Phase 5：旅遊業 Hotcake 完整化，12 週以上

目標：讓祕島成為導遊每天使用的營運系統。

進階功能：

1. 多導遊團隊。
2. 多語系行程頁。
3. 企業包團詢價。
4. 客戶分眾推播。
5. 評價邀請自動化。
6. 回訪行銷。
7. 進階營收報表。
8. 廣告投放後台。
9. API / Webhook。
10. 自訂網域。

成功指標：

- 付費導遊穩定成長。
- 導遊月留存 80% 以上。
- 曝光收入與 SaaS 收入並行。
- 平台自然流量可持續帶來新旅客。

## 10. 主要風險與對策

### 10.1 風險：導遊不用系統

原因可能是導遊習慣 LINE 手動接單。

對策：

- 初期提供代上架服務。
- 讓預約頁比 Google 表單漂亮很多。
- 保留 LINE 聯絡入口，不強迫導遊完全改流程。
- 先做最痛的功能：行程頁、預約、客戶名單、訂單提醒。

### 10.2 風險：沒有旅客流量，曝光賣不出去

對策：

- 初期不把曝光當主要收入。
- 先賣工具付費功能。
- 用導遊分享連結取得第一批流量。
- 用地區頁與主題頁做 SEO。

### 10.3 風險：免費用戶成本過高

對策：

- 限制圖片數量與容量。
- 限制免費版預約數、客戶數、行程數。
- 外部通知放付費版。
- 報表查詢範圍限制。

### 10.4 風險：付費曝光降低旅客信任

對策：

- 付費曝光需清楚標示。
- 排序不能完全由付費決定。
- 評價、回覆速度、可預約程度仍應影響排序。
- 對低品質行程保留人工審核與下架權限。

## 11. 最終建議

祕島的最佳 10/10 計畫是：

```txt
免費導遊工具
  → 導遊導入自己的旅客
  → 累積行程與內容
  → SEO 與自然流量成長
  → 工具進階功能收費
  → 分類頁 / 搜尋 / 主題專區曝光收費
  → 成為旅遊業 Hotcake
```

最重要的原則：

> 先讓導遊每天需要祕島，再讓旅客透過祕島找到導遊，最後才讓前台曝光變成可販售資產。
