# 01. 專案計劃書

## 1.1 問題定義

### 導遊端（供給側）
- 接案管道分散：靠 Line 群、口耳相傳、旅行社仲介
- 金流麻煩：LINE Pay、現金、轉帳各自為政
- 沒有統一排班與行程管理工具
- 旅行社抽成高（30~40%），收入不穩

### 遊客端（需求側）
- KKday / Klook 以「標準化商品」為主，缺乏在地客製感
- 找不到可信任的在地小團導遊
- 無法事先了解導遊背景、評價

## 1.2 解決方案

**雙邊平台：**
1. **消費者側** — 瀏覽活動、選擇導遊、預約日期、線上付款
2. **導遊側** — 上架行程、接收訂單、管理排班、自動結算收款

## 1.3 目標族群

| 族群 | 描述 | 規模估算 |
|------|------|---------|
| 主力遊客 | 25~45 歲、FIT 自由行、願意為體驗付費 | 台灣每年入境旅客約 1,200 萬 |
| 導遊 | 有導遊證或在地達人、願意接散客 | 台灣持證導遊約 3 萬人 |

## 1.4 核心 KPI（12 個月）

| 指標 | M3 | M6 | M12 |
|------|----|----|-----|
| 上架活動數 | 30 | 100 | 300 |
| 活躍導遊數 | 10 | 40 | 100 |
| 月訂單數 | 50 | 200 | 500 |
| 月 GMV | NT$75K | NT$300K | NT$750K |
| 月平台收入（15%） | NT$11K | NT$45K | NT$112K |

## 1.5 里程碑

### Phase 1 — 手動驗證（Month 1~2）
- [ ] 招募 10 位導遊上架活動（Google Form）
- [ ] 用 Notion + Line 手動接 50 筆訂單
- [ ] 驗證遊客願付意願 + 導遊操作意願
- [ ] 收集 20 份用戶訪談

### Phase 2 — MVP 建構（Month 2~4）
- [ ] 前端：活動列表 + 活動頁 + 預約流程
- [ ] 金流：ECPay 信用卡 + ATM 代碼
- [ ] 後台：基本訂單管理
- [ ] 上線 Beta 測試（邀請制）

### Phase 3 — 導遊後台（Month 4~6）
- [ ] 導遊儀表板：訂單、行事曆、收益報表
- [ ] 自動結算 + 提款申請
- [ ] 評價系統
- [ ] LINE Pay 串接

### Phase 4 — 成長（Month 6~12）
- [ ] SEO 優化
- [ ] 社群行銷啟動
- [ ] 企業包團（B2B）功能
- [ ] 天使輪募資

## 1.6 風險評估

| 風險 | 可能性 | 衝擊 | 對策 |
|------|--------|------|------|
| 導遊不願使用數位工具 | 中 | 高 | 提供手把手教學 + Line 輔助接案 |
| KKday 跟進推出類似功能 | 高 | 中 | 加速在地化 + 導遊黏著度 |
| 金流審核被拒 | 低 | 高 | 同時申請 ECPay + 藍新備援 |
| 消費者信任不足 | 中 | 高 | 強化導遊 KYC 認證 + 評價系統 |
