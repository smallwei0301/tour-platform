# 導遊入駐 SOP

> 最後更新：2026-04-20
> 狀態：Draft v1

## 1. 目的
建立導遊從申請、審核、上架到可接單的標準流程，降低人工 onboarding 混亂。

## 2. Onboarding 流程
1. 導遊提交申請
2. 平台進行基本資格審核
3. 補件 / 退件 / 通過
4. 建立導遊後台帳號或邀請連結
5. 協助完成第一個活動上架
6. 確認付款 / 結算 / 客服規則已理解
7. 正式開放接單

## 3. 建議必填資料
- 姓名 / 品牌名稱
- 聯絡方式
- 可服務地區
- 專長主題
- 導覽經驗
- 身分驗證 / 證照（若適用）
- 收款帳戶資訊（可後補）

## 4. 審核結果分類
### 通過
- 基本資料齊全
- 主題與平台定位相符
- 能提供可售體驗內容

### 補件
- 缺基本介紹 / 行程內容 / 圖片
- 證照或身份資料不完整

### 退件
- 主題與平台定位不符
- 服務內容有法規風險
- 無法證明基本可信度

## 5. 通過後的第一週目標
- 完成導遊帳號啟用
- 完成第一個活動資料建立
- 補齊封面 / 圖片 / 行程細節
- 確認退款 / 結算 / 接單規則理解

## 6. 與文件的連動
- 導遊需閱讀 dashboard 操作手冊
- 導遊需理解退款政策與結算規則
- 導遊需知道客服 / 緊急事件回報窗口

## 7. 後續待補
- 歡迎信範本
- 補件通知範本
- 退件通知範本
- KYC 文件保存規則
