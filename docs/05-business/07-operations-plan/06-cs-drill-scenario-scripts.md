## 情境 1：旅客取消訂單

### 角色設定
- **客服（Wei）**：接聽電話 / LINE
- **旅客（模擬）**：訂了「高雄柴山探洞」，想取消，出發日是明天

### 腳本

**旅客：** 你好，我想取消我的訂單，我明天有訂一個柴山的活動。

**客服：** 您好！請問您方便提供您的姓名或預約時的 Email，讓我查一下訂單？

**旅客：** 王小明，Email 是 test@example.com

**客服：** 好的，我找到了訂單。您目前的出發日是明天，依照我們的取消政策：出發前 72 小時內取消，將不退款。請問您確定要取消嗎？

**旅客：** 啊…真的嗎？可不可以例外處理？

**客服：** 我完全理解您的不便。因為導遊需要事前準備，這個規定是固定的。如果您是因為不可抗力（例如颱風警報），請告訴我，我幫您確認是否符合例外條件。

**旅客（路徑 A — 接受）：** 好吧，我了解，請幫我取消。

**客服：** 好的，我現在幫您取消。您的訂單已取消，因為在 72 小時內，這次無法退款。我會發 Email 確認給您，請注意收信。

**旅客（路徑 B — 不可抗力）：** 我是因為颱風警報。

**客服：** 如果今天有發布颱風警報，我們會主動聯繫您安排退款或改期，請問您看到哪個警報？我來查一下。（查詢中央氣象署）

---

### 📌 操作步驟（演練時同步做）
1. 登入 Admin POS → 搜尋訂單
2. 確認 `booking_status` / `payment_status`
3. 查 `refund_policy.md` 計算可退金額
4. 手動觸發退款（Admin → 訂單 → 退款）
5. 記錄到 ops log

## 情境 2：退款異常

### 角色設定
- **客服（Wei）**：接 Email
- **旅客（模擬）**：7 天前申請退款，但還沒入帳

### 腳本

**旅客 Email：** 你好，我 7 天前申請退款（訂單編號 TP-00023），但到現在還沒有收到退款，請問怎麼回事？

**客服（初次回覆 — 2 小時內）：**
> 王小明您好，感謝您聯繫我們。
>
> 我們已查到您的訂單 TP-00023，退款已於 [日期] 申請並送出。一般信用卡退款需要 5–7 個工作日由銀行處理，有時會拉長到 10 天。
>
> 請問您的退款是退回信用卡還是電子帳戶？如果已超過 10 個工作天，請告知，我們會進一步向金流商確認。
>
> 如有急需，我們可協助您確認 ECPay 退款狀態。
>
> 祝安好，Midao 客服

**旅客（後續）：** 已經 10 天了，信用卡。

**客服（查 ECPay + 升級）：**
1. 登入 ECPay 商家後台 → 退款記錄
2. 確認退款是否「送出」（ECPay 狀態）vs「入帳」（銀行端）
3. 若 ECPay 顯示成功 → 請旅客聯繫發卡銀行，提供退款交易日期和金額
4. 若 ECPay 顯示失敗 → 立即升級給 Wei，手動重新申請退款

## 情境 3：出團異常（不可抗力）

### 角色設定
- **客服（Wei）**：接到導遊 LINE
- **導遊（Andy Lee）**：今天颱風，無法出團
- **3 名旅客已預約**

### 腳本

**導遊 LINE：** Wei 你好，今天下午的柴山行程因為大雨和颱風警報，我判斷不安全，要取消，請你幫我通知旅客。

**客服（立即處理 — 出發前 4 小時前）：**

**步驟 1：確認天氣狀況**
- 查中央氣象署是否已發布颱風 / 豪雨 / 大雨特報
- 截圖存檔（作為不可抗力證明）

**步驟 2：查詢所有當日訂單**
- Admin → 搜尋今日活動 → 匯出旅客聯絡名單

**步驟 3：逐一聯繫旅客（SMS + Email + LINE）**
> 王小明您好，由於今日颱風警報，柴山探洞（今日下午 14:00）因安全考量取消出團。
>
> 您的訂單將全額退款，預計 5–7 個工作日入帳。如需改期，請回覆此訊息，我們協助您安排。
>
> 不便之處，我們非常抱歉。Midao 客服

**步驟 4：觸發退款**
- Admin POS → 逐筆全額退款（不可抗力 → 全額）
- 備注欄填寫：「颱風取消 - 導遊申請 - 全額退款」

**步驟 5：補償導遊**
- 依 `docs/05-business/06-payment-plan/05-settlement-payout-ops-runbook.md` 確認導遊誤工費政策
- 記錄到 ops log

## 情境 4：緊急安全事故

### 角色設定
- **客服（Wei）**：接到導遊緊急電話
- **導遊（Andy Lee）**：出團中，有旅客在柴山扭傷腳踝

### 腳本

**導遊電話：** Wei！緊急情況，有一個旅客在山上扭傷腳踝，她說痛到不能走路，我需要怎麼辦？

**客服（第一時間）：**

**步驟 1：確認人身安全**
- 旅客意識清楚嗎？
- 傷勢是否需要立即送醫（骨折 / 開放性傷口 / 失去意識 → 立刻叫 119）
- 目前確切位置？（GPS / 路標）

**步驟 2：視傷勢決策**

*情況 A — 輕傷（扭傷，可以緩步行走）：*
- 導遊協助旅客緩步下山
- 在山下安排旅客叫計程車或家人接送
- 建議旅客之後就醫留下記錄
- 記錄事故到 ops log

*情況 B — 中重傷（無法行走 / 懷疑骨折）：*
- 立刻打 119，說明位置「壽山柴山步道，座標 [GPS]」
- 陪伴旅客在原地等待
- 通知緊急聯絡人（旅客填寫的）
- Wei 立刻電話告知保險顧問聯絡人

**步驟 3：事後處理**
- 48 小時內完成 incident report（使用 `04-incident-response.md §5.5 模板`）
- 與旅客確認是否需要理賠協助
- 若旅客有損失，升級到法律顧問確認賠償

**緊急聯絡鏈（演練時需確認這些人真的聯絡得到）：**
- 119（緊急救援）
- 旅客緊急聯絡人（訂單填寫）
- 保險顧問：待確認（#797 D.3）
- 法律顧問：待確認（#797 D.1）
- Wei 備用電話：填入 `11-emergency-contact-chain.md`
