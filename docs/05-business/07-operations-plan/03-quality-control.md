# 活動品質控制手冊

> 最後更新：2026-04-20
> 狀態：v2 pre-launch 可執行版本

## 1. 目的
建立 Tour Platform 對活動品質、導遊表現與上架內容的一致審核標準，避免體驗品質失控。

## 2. 活動上架審核重點
- 標題是否清楚且不誤導
- 活動描述是否完整
- 圖片是否真實、清楚、足夠
- 價格是否合理且與內容相符
- 集合地點 / 時程 / 注意事項是否明確
- 取消規則是否一致

**執行者：** Admin / 平台操作員
**頻率：** 每次新活動上架前必做；soft-launch 期間每 48h 巡查一次
**通過標準：** 全部項目打勾，無 HOLD 項目
**紀錄格式：** [審核者] [日期] [活動 ID] [結果: PASS/HOLD/FAIL] [待修項]

### 活動上架品質 Checklist（Soft-launch v1）

**內容正確性**
- [ ] 標題 ≤ 30 字，清楚描述主要體驗
- [ ] 活動描述 ≥ 100 字，涵蓋行程、亮點、注意事項
- [ ] 圖片：至少 1 張封面（16:9）+ 1 張說明圖，無浮水印、無無關圖片
- [ ] 集合地點：有完整地址或 Google Maps 連結
- [ ] 時程：有明確出發/結束時間
- [ ] 適合人數：有 min/max 限制
- [ ] 安全注意事項：有明確提示（體力需求、年齡限制、裝備）

**價格與規則**
- [ ] 價格符合市場行情，有說明含/不含項目
- [ ] 取消規則：與退款政策 v2 一致（見 04-refund-policy-v2.md）
- [ ] 改期規則：有說明或依預設 v2 規則

**旅客期待管理**
- [ ] FAQ / 問答已有至少 2 則常見問題與解答
- [ ] 無誤導性文案（「保證最好」、「100% 安全」等絕對用語）

## 3. 導遊品質觀察面向
- 是否準時
- 是否與頁面描述一致
- 是否有安全風險
- 是否有高頻客訴
- 是否能穩定回覆平台需求

### 導遊品質 Gate（Soft-launch v1）

**回覆 SLA**
- [ ] 平台訊息：48h 內回覆
- [ ] 旅客問題（客服轉達）：24h 內回覆

**出團準備**
- [ ] 已確認行程、集合地點、天氣備案
- [ ] 緊急聯絡方式可用（電話、LINE 至少一種）
- [ ] 若取消或延遲，至少提前 24h 通知旅客

**內容一致性**
- [ ] 實際提供服務與活動頁描述一致（時間、地點、內容）
- [ ] 沒有未告知的額外收費

**風險標記**
- [ ] 高體力風險活動有適當說明
- [ ] 首次帶團的新導遊標記「新導遊觀察期」，前 3 團有 Admin 關注

**Evidence 格式：** [審核者] [日期] [導遊 ID] [結果: PASS/觀察/限制/暫停] [備注]

## 4. 品質處置分級
### 正常
- 維持上架

### 觀察
- 有零星問題，先提醒改善

### 限制
- 暫停新上架 / 暫停部分活動

### 下架 / 停權
- 嚴重客訴
- 安全事故
- 持續不改善

### Soft-launch 通過/HOLD 標準

| 情境 | 處置 | 連動控制 |
|------|------|---------|
| 活動內容有誤導性 | HOLD（不上架）| 人工 Admin override |
| 導遊失聯 > 4h | 緊急升級 P1 | 通知 CS，考慮暫停其活動 |
| 安全事故 | P0/P1 升級 | 啟用 `new_booking_paused` (#506) |
| 旅客大量投訴 | 評估限制 | 升 Go/No-Go HOLD (#505) |
| 金流/付款異常 | P1 升級 | 啟用 `refund_manual_only` (#506) |

參考：#529 incident response、#506 kill-switch、#505 Go/No-Go

## 5. 定期巡查建議
- 每月檢查一次活動頁內容
- 每月回顧退款 / 客訴 / 異常取消紀錄
- 每月標記高風險導遊與高風險活動

## 6. 黑名單 / 高風險名單
需明確記錄：
- 原因
- 日期
- 誰做判定
- 是否可申訴
- 何時複審

## 7. 與其他 SOP 的連動
