## 首次登入（邀請碼）

### 步驟一：取得邀請連結

平台管理員會透過邀請碼機制產生專屬連結傳給你，格式如下：

```
https://[平台域名]/guide/login?token=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
```

> ⚠️ **注意：** 邀請連結有效期 **24 小時**，且只能使用 **一次**。

### 步驟二：設定密碼

1. 點擊邀請連結
2. 進入「設定你的登入密碼」畫面
3. 輸入密碼（至少 6 個字元）
4. 確認密碼（再輸入一次）
5. 點「設定密碼並登入」

✅ 成功後自動進入儀表板

### 密碼規則

- 最少 6 個字元
- 無其他限制，建議使用英數混合

## 一般登入與登出

### 登入

1. 前往 `/guide/login`
2. 輸入你的 **導遊 ID**（UUID 格式，可向管理員索取）
3. 輸入密碼
4. 點「登入」

> 💡 **導遊 ID 在哪裡找？** 首次登入成功後，ID 存在瀏覽器 cookie 中（`guide_id`）；或聯絡管理員確認。

### 登出

點右上角 Navbar 中的「**登出**」按鈕，系統會清除 session 並返回登入頁。

### Session 時效

Session 有效期為 **7 天**。超過後需重新登入。若忘記密碼，請聯絡管理員重新產生邀請碼。

## 儀表板

進入後台後的預設頁面。

```
/guide/dashboard
```

### 頁面內容

#### 統計卡片（上方）

| 卡片 | 說明 |
|------|------|
| 本月預訂數 | 當月旅客成功付款的訂單數量 |
| 近期訂單 | 最近有訂單的場次數量 |
| 本週場次 | 本週七天內有開放的場次數量 |

#### 近期訂單列表

顯示最近幾筆旅客訂單，欄位包含：

- 旅客姓名
- 行程名稱
- 人數
- 訂單狀態（確認中 / 已確認 / 已取消）

#### 本週場次列表

顯示本週的場次，欄位包含：

- 日期
- 方案名稱
- 已訂 / 總容量（例：`4/8`）
- 場次狀態（開放 / 已關閉 / 額滿）

## 場次管理

```
/guide/schedules
```

### 頁面內容

列出你旗下所有行程的場次，可依狀態篩選。

### 篩選頁籤

| 頁籤 | 顯示內容 |
|------|---------|
| 即將出發 | 尚未開始、且狀態為「開放」的場次 |
| 全部 | 所有場次 |
| 已結束 | 日期已過的場次 |

### 開啟 / 關閉場次

1. 找到目標場次
2. 點右側 **「關閉」** 按鈕 → 場次狀態變為「已關閉」，旅客無法預訂
3. 點 **「開啟」** 可恢復

> ⚠️ 關閉場次 **不會取消** 已成立的訂單，只影響是否接受新預訂。

### 修改容量

1. 找到目標場次的容量數字（有虛線底線）
2. 點擊數字 → 變成可輸入的欄位
3. 輸入新的容量（整數）
4. 按 **Enter** 或點擊欄位外側確認

> ⚠️ **規則：** 新容量不能低於已訂人數。
>
> 例：已有 4 人訂位，容量不能設為 3 或以下。

## 訂單查看

```
/guide/bookings
```

### 頁面內容

列出你旗下行程的所有旅客訂單。

### 訂單列表欄位

| 欄位 | 說明 | 隱私保護 |
|------|------|---------|
| 旅客姓名 | 旅客本名 | 無遮罩 |
| 行程 | 行程名稱 | — |
| 場次日期 | 預定出發日 | — |
| 人數 | 預訂人數 | — |
| Email | 旅客信箱 | **隱碼顯示**（j\*\*\*@gmail.com） |
| 狀態 | 訂單狀態 | — |

### 篩選訂單

點上方篩選按鈕，可依狀態篩選：
- **全部**
- **已確認**（`confirmed`）
- **待付款**（`pending_payment`）
- **已取消**（`cancelled_by_user` / `cancelled_by_guide`）

### 查看訂單詳情

點擊任一訂單（整列）或「詳情」按鈕，彈出詳細視窗：

| 欄位 | 說明 |
|------|------|
| 旅客姓名 | 旅客本名 |
| **完整電話號碼** | 明碼顯示，方便出發前聯絡 |
| Email | 隱碼格式（j\*\*\*@gmail.com） |
| 場次日期 | 預定出發日期 |
| 人數 | 預訂人數 |
| 訂單金額 | 已付金額 |
| 訂單狀態 | 目前狀態 |

> 💡 電話號碼右側有「📋 複製」按鈕，方便快速複製。

## 常見問題

### Q：忘記密碼怎麼辦？
A：目前系統不支援自助重設密碼，請聯絡平台管理員重新產生邀請碼，使用新邀請連結設定新密碼。

### Q：邀請碼連結失效了？
A：邀請碼有效期 24 小時且只能使用一次。過期或已使用後，請聯絡管理員重新產生。

### Q：為什麼看不到旅客的完整 Email？
A：基於個人資料保護，訂單列表中的 Email 以隱碼方式顯示。如需聯絡旅客，建議使用詳情頁中的完整電話號碼。

### Q：關閉場次後，已訂位的旅客會被取消嗎？
A：不會。關閉場次僅阻止新預訂，已成立的訂單不受影響。如需取消訂單，請聯絡管理員處理。

### Q：容量上限可以設成 0 嗎？
A：不行。容量最低為已訂人數，且必須 ≥ 1。

### Q：後台只能看到自己的場次和訂單嗎？
A：是的。系統會自動隔離資料，你只能查看與自己 guide_id 相關的場次和訂單。

### Q：Session 多久會過期？
A：7 天。過期後需重新輸入 Guide ID 和密碼登入。

## 隱私與安全說明

| 機制 | 說明 |
|------|------|
| 密碼儲存 | SHA-256 + salt 雜湊，永不明文儲存 |
| Session Token | HMAC-SHA256 簽章，防偽造 |
| Cookie 安全 | HttpOnly（防 XSS）+ Production 環境 Secure 旗標（僅 HTTPS） |
| 資料隔離 | 所有 API 均驗證 ownership，非本人資源回傳 403 |
| Email 保護 | 旅客 Email 在導遊端自動隱碼顯示 |
