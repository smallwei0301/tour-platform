# worklogs — 任務記憶錨點（issue＋repo 雙寫的 repo 側）

每個進行中的 issue 一個檔：`issueNNNN.md`，模板見 `.cursor/harness/04_templates.md` §W。

規則（`.cursor/harness/02_orchestration.md` §4）：
- 開工先讀（或建立）worklog；context 壓縮/恢復後第一件事是重讀它。
- 每個里程碑：更新 worklog ＋ 同步精簡留言到 GitHub issue（模板 §I）。
- 「絕不重做」區是防迷航的核心——動手改檔前先查。
- 任務關閉（issue closed）後，worklog 保留作歷史，不刪除。
